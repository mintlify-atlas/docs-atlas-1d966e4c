Welcome to **Recetas Deliciosas Como Servicio (RDCS)**, a web platform where chefs share exclusive recipes through a secure subscription model. Every recipe is encrypted with AES-128-GCM and only subscribers with an active contract can decrypt and read the content — even the server operator cannot access recipes without authorization.

## How it works

<Steps>
  <Step title="Create an account">
    Register as a **chef** or **subscriber**. During registration, an ECDSA P-256 key pair is generated for you. Your private key is encrypted with your password using PBKDF2 + AES-128-GCM before being stored — only you can unlock it.
  </Step>
  <Step title="Confirm your email">
    A confirmation link is sent to your inbox via a 32-byte random token. Click it to activate your account.
  </Step>
  <Step title="Chefs publish, subscribers subscribe">
    Chefs upload recipe content which is automatically encrypted with a unique AES-128-GCM key before being stored in the external vault. Subscribers browse the recipe catalog and purchase subscriptions (3, 6, or 12 months) to their chosen chef.
  </Step>
  <Step title="Access encrypted content">
    Subscribers with an active contract can request decrypted recipe content. The platform verifies the contract's expiry date before returning the plaintext — once a subscription lapses, access is automatically revoked.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Hybrid cryptography" icon="lock">
    ECDSA P-256 identity keys protected per-user with PBKDF2 + AES-128-GCM. No master key — only your password unlocks your identity.
  </Card>
  <Card title="Encrypted recipe vault" icon="shield">
    Recipe files are stored as `.enc` files in an external vault. Each recipe has a unique AES-128-GCM key. SHA-256 hashes verify file integrity.
  </Card>
  <Card title="Time-limited access" icon="clock">
    Subscription contracts enforce 3, 6, or 12-month access periods. Decryption is denied automatically once a contract expires.
  </Card>
  <Card title="Zero-knowledge partial model" icon="eye-slash">
    The server stores encrypted private keys and ciphertexts only. Neither the server operator nor network observers can read recipe content.
  </Card>
</CardGroup>
